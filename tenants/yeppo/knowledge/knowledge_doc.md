MANEJO DE RECLAMOS
Cuando alguien reclama por un pedido que no llegó o llegó mal, primero validamos su molestia con frases como "entiendo tu frustración :c" o "qué lata". Luego investigamos: pedimos número de pedido, revisamos el estado, y si es un error nuestro ofrecemos soluciones. Si el error es de tienda (producto equivocado), se escala al área y se evalúa una excepción como cambio directo, aunque por política no se haga. Si el cliente insiste, derivamos internamente para resolver. En caso de demora con el courier, contactamos a logística para agilizar. Siempre mantenemos la paciencia, incluso cuando el cliente escribe insistentemente. No prometemos fechas si no podemos cumplirlas. Si un cliente pide cancelar, lo hacemos y explicamos que el reembolso tarda lo que demore el banco. No tomamos la culpa personal, pero nos disculpamos en nombre del equipo. Para problemas con mercadopago o web, derivamos al área correspondiente.

CUANDO DERIVAR A HUMANO
Derivamos a un agente humano cuando el cliente necesita confirmación de stock en tiempo real en tienda física, seguimiento detallado de un pedido, cancelación o cambio porque el bot no tiene acceso a modificar pedidos. También cuando la situación es delicada (reclamo por producto defectuoso, error de despacho, problema con el pago, queja por mal servicio). Si el cliente pide una rutina muy específica que no puede resolver el flujo automático, se deriva para asesoría personalizada. Para mayoristas siempre derivamos al correo de Juan. Para colaboraciones, al WhatsApp de marketing. En general, si la conversación excede las capacidades del bot o se nota frustración alta, se pasa a humano.
